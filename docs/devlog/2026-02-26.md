## 1. 검색창 키보드 단축키

### 변경 파일
- `frontend/components/search/SearchAutocomplete.tsx`

### 구현 내용

| 단축키 | 동작 |
|--------|------|
| `/` | 검색창 포커스 + 드롭다운 열기 |
| `Esc` | 검색창 blur + 드롭다운 닫기 |

#### 추가된 useEffect
```tsx
useEffect(() => {
  const handleKeyDown = (e: KeyboardEvent) => {
    const tag = (e.target as HTMLElement).tagName;
    if (e.key === "/" && tag !== "INPUT" && tag !== "TEXTAREA") {
      e.preventDefault();
      inputRef.current?.focus();
      setIsOpen(true);
    } else if (e.key === "Escape") {
      inputRef.current?.blur();
      setIsOpen(false);
    }
  };
  document.addEventListener("keydown", handleKeyDown);
  return () => document.removeEventListener("keydown", handleKeyDown);
}, []);
```

#### UI 힌트 뱃지
- 검색어 없을 때 input 우측에 `kbd` 뱃지 표시 (`/`)
- `sm` 이상 화면에서만 노출, `pointer-events-none`
- 검색어 입력 시 X 버튼으로 교체

### 설계 결정
- `tagName` 체크: input/textarea 안에서 `/` 입력은 정상 동작 유지
- `contentEditable` 미체크 (RecFlix에 해당 요소 없음)
- 전역 리스너 1개로 완결, 별도 훅 불필요
