### `frontend/app/movies/page.tsx`

| 항목 | 변경 내용 |
|------|----------|
| 영화 목록 | `useQuery(['movies', query, selectedGenre, sortBy, currentPage])` — currentPage queryKey 포함으로 자동 재요청 |
| 장르 목록 | `useQuery(['genres'])` + `staleTime: Infinity` |
| 추천 영화 | `useQuery(['popularMovies'])` + `enabled: movies.length === 0 && !!query && !isLoading` |
| 무한 스크롤 | `useInfiniteQuery(['movies-infinite', ...])` + `initialPageParam: 1` |
| 페이지네이션 | `fetchPage` 중복 람다 완전 제거 — `setCurrentPage`만으로 `useQuery` 자동 재요청 |
| 상태 | `loading` → `isLoading`, `loadingMore` → `isFetchingNextPage` |
| 필터 초기화 | `useEffect`로 query/genre/sort 변경 시 `currentPage` → 1 리셋 |

**핵심 패턴**: 페이지네이션 + 무한 스크롤 듀얼 모드 → 각각 별도 훅(`useQuery`, `useInfiniteQuery`), `enabled` 플래그로 활성화 모드만 실행

## 2. zustand 무한 루프 패턴 수정

### 문제
zustand store에서 꺼낸 함수(`fetchUser`, `fetchInteraction` 등)는 **매 렌더마다 참조가 새로 생성**됨.
이 함수를 `useEffect` 의존성 배열에 넣으면 → 렌더 → 참조 변경 → useEffect 실행 → 렌더 → ... **무한 루프**

### 해결 패턴
```tsx
useEffect(() => {
  if (isAuthenticated) fetchUser();
  // eslint-disable-next-line react-hooks/exhaustive-deps
}, [isAuthenticated]); // fetchUser 제거 — zustand 함수 참조 변경으로 인한 무한 루프 방지
```

### 수정된 파일
| 파일 | 제거한 deps |
|------|------------|
| `app/profile/page.tsx` | `fetchUser`, `router` |
| `app/movies/[id]/page.tsx` | `fetchInteraction` |
